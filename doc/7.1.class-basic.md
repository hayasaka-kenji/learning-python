# クラス

## クラスの定義
```py
class Person(object):
    def say_something(self):
        print('hello')

person = Person()
person.say_something()
```
- `(object)`はpython3から不要になったが、ベースクラスとして明示することができる。

## 初期化
```py
class Person(object):
    def __init__(self, name):
        self.name = name
        print(self.name)

    def say_something(self):
        print('I am {}. Hello.'.format(self.name))
        # 自身の関数を使う
        self.run(10)

    def run(self, num):
        print('run ' * 10)

person = Person('Mike')
person.say_something()
```
- `__init__`はクラスを生成したときに読まれる。`self`はデフォルトで持たせるようにする。
- 自身の関数を使うときは`self`を用いる。

## コンストラクタ、デストラクタ
```py
class Person(object):
    # コンストラクタ
    def __init__(self, name):
        self.name = name
        print(self.name)

    # デストラクタ
    def __del__(self):
        print('good bye')
```
- 実行が終わるとクラスが削除され、デストラクタを使用することができる。
- 実行されるタイミングを指定したいとはdel関数を使用する。

## クラスの継承
```py
class Car(object):
    def run(self):
        print('run')

class ToyotaCar(Car):
    pass

class TeslaCar(Car):
    def auto_run(self):
        print('auto run')

toyota_car = ToyotaCar()
toyota_car.run()
# run

tesla_car = TeslaCar()
tesla_car.run()
tesla_car.auto_run()
# run
# auto run
```
`pass`はクラスの作成するが、なにも定義しないときに使う。

### 親クラスの要請
```py
class Car(object):
    def __init__(self, model=None):
        self.model = model
```
コンストラクタを書き換える際に、親クラスの内容を引き継ぐときは`super()`を使う。

## プロパティ
```py
class TeslaCar(Car):
    def __init__(self, model='Model S',
                 enable_auto_run=False,
                 passwd='123'):
        # 親クラスの要請
        super().__init__(model)
        self._enable_auto_run = enable_auto_run
        self.passwd = passwd

    # 読み込み専用にする
    @property
    def enable_auto_run(self):
        if self.passwd == '456':
            return self._enable_auto_run
        else:
            raise ValueError

    # 書き込み可能にする
    @enable_auto_run.setter
    def enable_auto_run(self, is_enable):
        self._enable_auto_run = is_enable
```

- クラス定義ではないところで値を変えてほしくないときに使う。
- `__`がある場合はクラス内からはアクセスできるが、外からは不可。
- `_`はアクセスできるが明示的に、アクセスしないものとしている。
