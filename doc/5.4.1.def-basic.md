# 関数定義
```py
def say_something():
    s = 'hi'
    return s
result = say_something()
print(result)
# hi
```
- `def`で始めて`()`で引数を渡す。
- 関数定義をしてから、関数を呼び出す。`()`がないと、実行されないので注意。
- 型は`function`

## 引数
```py
def what_is_this(color):
    if color == 'red':
        return 'tomato'
    elif color == 'green':
        return 'cabbage'
    else:
        return "I don't know"

result = what_is_this('yellow')
print(result)
# I don't know
```

### 引数の型
```py
# 引数と返り値の型を指定する
def add_num(a:int, b:int) -> int:
    return a + b

print(add_num(10, 20))
# 30

print(add_num('a', 'b'))
# ab エラーとして返ってこない。
```
- 引数や返り値を型固定することは可能だが、エラーとして返ってこないので注意する。

### 位置引数
```py
def menu(entree, drink, dessert):
    print(entree)
    print(drink)
    print(dessert)
# 順序を正しく入れないと期待していない結果になる
menu('beef', 'soda', 'cake')
```

### キーワード引数(キーワードアーギュメント)
```py
menu(entree= 'beef', drink='soda', dessert='cake')

# 混ぜてつかうことも可能
menu('beef', drink='soda', dessert='cake')
```

### デフォルト引数
```py
def menu(entree='beef', drink='soda', dessert='cake'):
    print('entree =>', entree)
    print('drink =>', drink)
    print('dessert =>', dessert)
```
- 位置引数とキーワード引数を混ぜて使うことも可能だが、位置を間違えると、エラーになる。

### デフォルト引数で気をつけること
```py
def test_func(x, l = []):
    l.append(x)
    return l

r = test_func(100)
print(r)
# [100]

r = test_func(100)
print(r)
# [100, 100]
```
- 関数の引数で定義したリストも参照渡しになる。
- Pythonはデフォルト引数でリストを渡すとバグに繋がりやすい。

```py
def test_func(x, l = None):
    if l is None:
        l = []
    l.append(x)
    return l

r = test_func(100)
print(r)
# [100]

r = test_func(100)
print(r)
# [100]
```
- 関数内で空の引数を作るようにすると、解決できる。
