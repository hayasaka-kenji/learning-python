# エラーハンドリング
```py
# 例外処理
l = [1, 2, 3,]
i = 5

try:
    () + l
except IndexError as exc:
    print('Dont worry: {}'.format(exc))
except NameError as exc:
    print(exc)
# 上記以外のエラーを取得する
except Exception as ex:
    print('other:{}'.format(ex))

print('last')
# other:can only concatenate tuple (not "list") to tuple
# last
```
(すべての例外を処理するような書き方は非推奨)

## 例外のクラス階層
https://docs.python.org/ja/3/library/exceptions.html#exception-hierarchy

## finaly, else
```py
try:
    l[0]
except IndexError as exc:
    print('Dont worry: {}'.format(exc))
else:
    print('done')
finally:
    print('clean up')
```
- `finally`はエラー関係なく実行される。
- `else`は問題なく終了した際に使用される。

## 独自例外
```py
raise IndexError('test error')
```
`raise`を使えばエラーを発生することができる。
